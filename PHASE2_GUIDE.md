# Phase 2: 影子镜像指挥中心 - 安装使用指南

## 🎉 新功能预览

Phase 2 引入了全新的**影子镜像指挥中心（Shadow Dashboard）**，让你可以：

- ✨ 在一个页面中以 Grid 布局（$m \times n$）同时观看多个模型的实时生成过程
- 🎨 享受统一、高颜值的 UI 设计，摆脱各平台杂乱的原生界面
- ⚡ 实时同步：流式输出的"影子"在 Dashboard 中实时跳动
- 🎛️ 自定义布局：支持 1×2、2×2、1×3、3×1 等多种网格布局
- 📤 统一输入：在 Dashboard 中输入 Prompt，一键发送到所有模型

## 📦 安装说明

### 1. 更新扩展（如果已安装）

如果你已经安装了 Phase 1 版本：

1. 打开 `chrome://extensions/`
2. 找到 "Multi-LLM Prompt Dispatcher"
3. 点击「重新加载」按钮 🔄

### 2. 首次安装

按照 README.md 中的标准安装流程即可。

## 🚀 使用 Dashboard (影子镜像指挥中心)

### 方式一：快捷键打开（推荐）

按下 **`Ctrl + Shift + D`** (Windows/Linux) 或 **`Cmd + Shift + D`** (Mac)

### 方式二：手动打开

1. 点击插件图标
2. 在 Popup 中点击「打开指挥中心」按钮（即将添加）
3. 或直接访问：`chrome-extension://YOUR_EXTENSION_ID/dashboard/dashboard.html`

---

## 🎛️ Dashboard 功能说明

### 控制栏

| 功能 | 说明 |
|------|------|
| **布局选择器** | 切换 Grid 布局（1×2、2×2、1×3、3×1） |
| **输入 Prompt** | 打开统一 Prompt 输入面板 |
| **刷新** | 重新扫描已打开的 LLM 标签页 |
| **全屏** | 切换全屏模式（沉浸式对比） |

### 模型卡片

每个卡片代表一个 LLM 标签页：

- **顶部状态**：
  - 模型名称 + 图标
  - 生成状态（等待中/生成中/已完成）
- **内容区域**：
  - 实时显示最新的助手回答
  - 自动滚动到最新内容
- **交互**：
  - 点击卡片 → 跳转到对应的浏览器标签页
  - 生成中时顶部显示动画进度条

## 💡 使用场景示例

### 场景一：同时观察多个模型对同一问题的回答

```
1. 打开 ChatGPT、Claude、Gemini 三个标签页
2. 按 Ctrl+Shift+D 打开 Dashboard
3. 选择 1×3 布局（三个卡片横向排列）
4. 点击「输入 Prompt」按钮
5. 输入：「请用简洁的语言解释量子纠缠」
6. 点击「发送到所有模型」
7. 实时观看三个模型同时生成回答的过程！
```

### 场景二：长时间多轮对话监控

```
1. 发起第一轮提问
2. 在 Dashboard 中查看所有回答
3. 点击某个卡片，跳转到具体页面查看细节
4. 回到 Dashboard，发起第二轮追问
5. 继续在统一视图中对比观察
```

## 🔧 技术亮点

### 影子镜像技术

- **不使用 Iframe**：避开了各平台的安全头限制（X-Frame-Options）
- **实时捕获**：使用 `MutationObserver` 监听各页面的 DOM 变化
- **增量同步**：只传输文本数据，性能消耗极低
- **自定义渲染**：统一的 UI 组件，美观且可扩展

### 数据流

```
ChatGPT Tab (后台)
  ↓ [MutationObserver 捕获]
Content Script
  ↓ [chrome.runtime.sendMessage]
Background Service Worker
  ↓ [postMessage 广播]
Dashboard Page
  ↓ [更新 UI]
用户看到实时更新的卡片！
```

## ⚠️ 注意事项

### 1. DOM 选择器依赖

各平台的 UI 可能更新导致适配器失效。如遇问题：
- 刷新 Dashboard（F5）
- 重新加载 LLM 页面
- 更新插件版本

### 2. 性能建议

- 同时监听 2-4 个模型时性能最佳
- 超过 4 个模型可能导致浏览器卡顿
- 建议关闭不需要的标签页

### 3. 当前限制（Phase 2 验证版本）

- ✅ **已实现**：ChatGPT 输出监听
- ⏳ **待实现**：Claude、Gemini 输出监听（Phase 2 完成版）
- ⏳ **待实现**：Markdown 渲染（Phase 3）

## 🐛 故障排除

### 问题：Dashboard 无法打开

**解决**：
1. 确认快捷键未被其他扩展占用
2. 手动访问 Dashboard URL
3. 检查扩展是否正确加载

### 问题：卡片显示"暂无输出"

**可能原因**：
1. 该标签页还未发送任何 Prompt
2. 监听器未启动

**解决**：
1. 点击 Dashboard 的「刷新」按钮
2. 在对应标签页发送一个 Prompt

### 问题：内容不同步

**解决**：
1. 刷新 Dashboard (F5)
2. 重新加载受影响的 LLM 标签页
